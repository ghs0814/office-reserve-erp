🛑 Stage 1: 로그인 및 권한 라우팅 (Auth & Role)
[ ] 일반 사원(레벨 1)으로 로그인 시 상단 헤더에 [비품 재고 관리], [관리자 결재함] 메뉴가 보이지 않는지 확인.

[ ] 최고 관리자(레벨 5)로 로그인 시 상단 헤더에 모든 관리자 전용 메뉴가 정상적으로 노출되는지 확인.

[ ] 로그아웃 후 브라우저 주소창에 main.jsp나 managerApproval.do를 직접 입력했을 때, 세션 차단 기능이 작동하여 로그인 화면으로 강제 이동(튕김) 되는지 확인.

🏢 Stage 2: 오피스 예약 시스템 (동시성 및 유효성 검증)
[ ] 달력 클릭 시 오늘 날짜 이전(과거)의 날짜가 아예 선택되지 않도록 회색 처리(disabled) 되어 있는지 확인.

[ ] '오늘 날짜' 선택 시, 현재 시간보다 과거 시간 버튼이 클릭 방지되어 있는지 확인.

[ ] 시간 버튼 여러 개 클릭 시 중간이 빈 시간(예: 10:00, 12:00)을 고르면 "연속된 시간만 선택 가능" 알림창이 뜨며 차단되는지 확인.

[ ] 특정 시간 선택 후 [선택 시간 확보] 버튼 클릭 시, 우측에 5분 타이머가 노출되며 카운트다운이 시작되는지 확인.

[ ] 타이머가 도는 동안 [예약 확정]을 누르지 않고 브라우저를 새로고침 한 뒤 다른 방을 구경하다가 5분이 지났을 때, 내가 선점했던 시간이 다시 예약 가능 상태(활성화)로 돌아오는지 확인 (Ghost Data 자동 클린업 확인).

💻 Stage 3: 전자 결재 및 비품 대여 (5단계 결재 워크플로우)
[ ] [일반 사원] 비품 대여 신청 시, 대여 수량을 잔여 재고(REMAIN_COUNT)보다 높게 입력했을 때 HTML 유효성 검사에 막혀 전송되지 않는지 확인.

[ ] [일반 사원] 반납 예정일을 대여 시작일보다 이전(과거)으로 설정했을 때 자바스크립트 경고창이 뜨며 차단되는지 확인.

[ ] [1단계 결재자 로그인] 로그인 후 전사 기안 문서함에 들어갔을 때, 방금 사원이 올린 기안이 1단계 상태로 대기 중인지 확인.

[ ] 결재 상세 페이지에서 [승인] 클릭 시, 현재 결재 단계(CURRENT_STEP)가 2단계로 올라가고 결재자 서명란에 로그인한 사람의 이름과 날짜가 찍히는지 확인.

[ ] [5단계 결재자 로그인] 최종 5단계 승인 클릭 시, 해당 기안의 상태가 '승인대기'에서 '대여중'으로 변경되는지 확인.

[ ] 비품 재고 관리 페이지에 들어가서, 5단계 최종 승인이 완료된 비품의 잔여 수량(REMAIN_COUNT)이 신청 수량만큼 즉시 차감되었는지 확인 (트랜잭션 무결성 검증).

👤 Stage 4: 마이페이지 통합 관리 (상태 동적 변경)
[ ] 내 예약 내역에서 과거에 진행되었던 예약의 상태가 '예약완료'가 아닌 '이용 종료'로 동적 변환되어 표시되는지 확인 (취소 버튼 사라짐).

[ ] 내 비품 대여 내역에서 대여 기간이 지난 비품에 빨간색 배지로 '미반납' 상태가 출력되는지 확인.

[ ] 대여중 / 미반납 상태의 비품을 [반납 처리] 버튼을 눌러 반납했을 때 상태가 '반납완료'로 변경되는지 확인.

[ ] 관리자 재고 페이지로 이동하여, 방금 반납한 비품의 잔여 수량이 다시 원래대로(+ 복구) 증가했는지 확인.
