# OfficeReserveERP: 지능형 사내 자원 관리 시스템평면도 기반의 시각적 예약과 5단계 계층형 전자결재를 결합한 자바 웹 프로젝트입니다. 단순한 기능 구현을 넘어, 실무에서 발생할 수 있는 결재 프로세스와 자원 충돌 문제를 기술적으로 해결하는 데 중점을 두었습니다.## 1. 프로젝트 개요개발 기간: 2026.01 - 2026.04  핵심 목표: 직관적인 UI를 통한 회의실 예약 및 엄격한 승인 절차를 거치는 비품 대여 시스템 구축  주요 특징:실시간 AJAX 기술을 활용한 예약 시간 중복 방지  사용자 직급(Level)에 따른 유연한 5단계 전자결재 워크플로우  도면 이미지를 활용한 시각적 층별 예약 인터페이스  ## 2. 기술 스택 (Tech Stack)구분기술상세 활용 내용BackendJava 17, JSP/ServletMVC 패턴 기반의 비즈니스 로직 처리 및 데이터 정제DatabaseH2 / Oracle사원, 비품, 예약 정보를 관리하는 관계형 DB 설계  ConnectionHikariCP고성능 커넥션 풀을 통한 DB 연결 최적화 및 안정성 확보  FrontendJavaScript (AJAX)페이지 새로고침 없는 실시간 데이터 조회 및 검증  LibraryLombokGetter/Setter 자동화를 통한 코드 가독성 및 생산성 향상## 3. 핵심 기능 상세### ① 평면도 기반 실시간 회의실 예약 (AJAX)시각적 UI: 실제 사무실 도면(floor4_map.png) 위 좌표에 버튼을 배치하여 직관적으로 회의실을 선택합니다.  실시간 중복 검증: 사용자가 날짜를 변경할 때마다 AJAX 요청을 통해 서버에서 해당 일자의 예약된 시간을 조회합니다.  지능형 활성화: 이미 예약된 시간대는 선택 버튼을 실시간으로 비활성화(Disabled)하여 데이터 충돌을 사전에 차단합니다.  ### ② 5단계 계층형 전자결재 시스템직급별 동적 라인: 사원의 레벨(1~5단계)에 따라 결재 시작 지점과 거쳐야 할 승인 단계가 자동으로 계산됩니다.  1등급(사원): 2-3-4-5단계 순차 승인 대기5등급(최고관리자): 기안 즉시 최종 승인 처리 및 재고 반영결재 현황 시각화: 5개의 서명란(SIGN1~5)을 통해 결재 진행률을 실시간으로 모니터링할 수 있습니다.  ### ③ 비품 재고 관리 및 대여 서비스 (Core)실시간 재고 반영: 결재가 완료되어 대여 상태가 되면 비품의 잔여 수량이 자동으로 차감됩니다.  사원 마이페이지: 본인의 대여 이력과 예약 현황을 한눈에 관리하며, 즉시 반납 및 취소 처리가 가능합니다.  ## 4. 데이터베이스 설계 (ERD 요약)EMPLOYEE: 사번, 아이디, 비밀번호, 성명, 권한 레벨(empLevel)  EQUIPMENT: 비품 번호, 이름, 총 수량, 잔여 수량(remainCount)  RENTAL_HISTORY: 대여 번호, 상태(Status), 결재 단계(approvalStep), 서명란 1~5  RESERVATION: 방 번호, 예약일, 시작/종료 시간, 목적  ## 5. 핵심 로직: 직급별 결재 프로세스Java// RentProcessController 중략 - 사용자의 등급에 따른 상신 로직
