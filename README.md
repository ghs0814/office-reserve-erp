## 💡 프로젝트 개요

단순한 CRUD(생성/읽기/수정/삭제)를 넘어, **실무에서 발생하는 데이터 충돌(동시 예약) 문제**와 **수직적 조직의 결재 프로세스**를 기술적으로 해결하는 데 중점을 둔 프로젝트입니다.

- **개발 기간:** 2026.01 ~ 2026.04
- **핵심 목표:** 
  1. 사용자 친화적인 도면 기반 예약 시스템 구축
  2. 직급에 따라 동적으로 변하는 계층형 전자결재 시스템 구현
  3. 실시간 재고 연동을 통한 비품 관리 자동화

## 🛠 기술 스택 (Tech Stack)

### Backend
- **Java 17** : 객체 지향적 비즈니스 로직 구현
- **JSP / Servlet** : MVC2 패턴을 적용한 아키텍처 설계
- **Lombok** : DTO 보일러플레이트 코드 최소화 및 유지보수성 향상

### Database
- **Oracle / H2 DB** : 관계형 데이터베이스를 활용한 구조적 데이터 관리
- **HikariCP** : 고성능 커넥션 풀(Connection Pool) 적용으로 DB 연동 속도 및 안정성 최적화

### Frontend
- **HTML5 / CSS3 / JavaScript**
- **AJAX (Fetch API)** : 비동기 통신을 통한 실시간 예약 가능 시간 검증 및 화면 렌더링

## 🚀 핵심 기능 상세 (Key Features)

### 1. 🗺 평면도 기반 스마트 회의실 예약 (AJAX)
*텍스트 위주의 딱딱한 폼 대신, 실제 사무실 도면을 클릭하여 예약하는 직관적인 UI를 제공합니다.*

- **Interactive UI** : 도면(`floor4_map.png`) 위 회의실을 클릭하면 해당 방의 상세 스펙(수용 인원, 장비 유무)과 함께 예약 창으로 이동합니다.
- **실시간 중복 예약 차단** : AJAX 통신을 통해 사용자가 선택한 날짜의 '이미 예약된 시간대'를 실시간으로 조회하고, 해당 시간 버튼을 **비활성화(Disabled)** 처리하여 데이터 충돌을 원천 차단합니다.

### 2. 📝 5단계 계층형 전자결재 시스템 (Core Logic)
*수평적/수직적 조직 문화를 모두 반영할 수 있는 동적 결재 라인 알고리즘을 구현했습니다.*

- **직급별 맞춤형 워크플로우** : 로그인한 사용자의 권한 레벨(`empLevel: 1~5`)을 판별하여 결재 시작 단계를 자동으로 설정합니다.
  - **사원(1단계)** : 기안 시 2단계부터 5단계까지 순차적인 승인이 필요합니다.
  - **최고관리자(5단계)** : 기안 즉시 모든 단계가 패스되며 '대여중' 상태로 전환됩니다.
- **결재 상태 시각화** : `SIGN1~5` 컬럼을 활용하여 현재 문서를 누가 승인했는지, 어느 단계에 머물러 있는지 직관적인 대시보드 형태로 제공합니다.

### 3. 📦 실시간 연동 비품 관리 대시보드
- **재고 동기화** : 대여 결재가 최종 승인(`대여중`)되는 즉시 비품 마스터 테이블의 '잔여 수량'이 차감되며, 반납 시 복구됩니다.
- **관리자 전용 CRUD** : 관리자 권한 로그인 시 모달(Modal) 창을 통한 비품 수정 및 폐기 기능을 제공합니다.

<details>
<summary><b>1. 동적 결재 라인 설정 로직 (Click to Expand)</b></summary>
<br>

사용자의 직급에 따라 결재 단계를 유연하게 스킵하는 컨트롤러 로직입니다.

```java
// RentProcessController.java
int myLevel = loginEmp.getEmpLevel(); 

if (myLevel >= 5) {
    // 5등급(최고 관리자): 기안 즉시 최종 승인 처리 및 대여 시작
    dto.setStatus("대여중");
    dto.setApprovalStep(5);
    dto.setSign1(loginEmp.getEmpName());
    // ... sign5까지 본인 서명 처리
} else {
    // 1~4등급: 본인 등급 다음 단계부터 결재 시작 (승인 대기)
    dto.setStatus("승인대기");
    dto.setApprovalStep(myLevel + 1);
    
    // 본인 등급까지는 자동으로 기안자 서명 처리 완료
    if (myLevel >= 1) dto.setSign1(loginEmp.getEmpName());
    // ... myLevel에 맞춰 서명 처리
}
```
</details>

<details>
<summary><b>2. 실시간 예약 중복 방지 (AJAX)</b></summary>
<br>

비동기 통신으로 예약된 시간을 확인하고 버튼을 제어하는 프론트엔드 로직입니다.

```javascript
// reserve.jsp
resDateInput.addEventListener('change', function() {
    const selectedDate = this.value;
    
    // 서버에 선택한 날짜의 예약 완료된 시간 목록을 요청
    fetch('checkReservedTime.do?roomId=' + roomId + '&resDate=' + selectedDate)
        .then(response => response.json())
        .then(reservedTimes => {
            // 버튼 상태 초기화 후, 이미 예약된 시간과 일치하는 버튼 비활성화
            timeButtons.forEach(btn => {
                if (reservedTimes.includes(btn.getAttribute('data-time'))) {
                    btn.disabled = true;
                }
            });
        });
});
```
</details>

## ⚙️ 실행 방법 (How to Run)

1. 이 저장소를 로컬 컴퓨터로 클론(`Clone`)합니다.
2. `Oracle` 또는 `H2 DB`에 포함된 `sql.txt` 파일의 스키마 및 더미 데이터를 실행하여 테이블을 세팅합니다.
3. `src/main/java/com/office/util/DBConnection.java` 파일에서 본인의 환경에 맞게 DB 연결 정보를 수정합니다.
4. `Apache Tomcat (v9.0 권장)` 서버에 프로젝트를 올리고 실행합니다.
5. `http://localhost:8080/OfficeReserveERP/index.jsp` 로 접속하여 시스템을 시작합니다.

# #Controller

# AdminEqListController.java

**파일 설명:** 관리자 전용 비품 재고 관리 페이지를 위한 데이터를 준비합니다. 데이터베이스에서 모든 비품의 목록(총 수량, 잔여 수량 등)을 조회하여 관리 대시보드 화면으로 전달합니다.

# ApprovalProcessController.java

**파일 설명:** 관리자 결재함에서 들어온 승인 또는 반려 요청을 처리합니다. 다단계 결재 로직에 따라 현재 결재 단계를 업데이트하고, 최종 단계일 경우 대여 상태를 '대여중'으로 변경하는 핵심 비즈니스 로직을 포함합니다.

# CancelReserveController.java

**파일 설명:** 회의실 예약 취소 요청을 처리하는 컨트롤러입니다. 사용자가 마이페이지 등에서 취소 버튼을 눌렀을 때 해당 예약의 번호를 받아 데이터베이스에서 상태를 '취소됨'으로 변경하는 역할을 합니다.

# CheckReservedTimeController.java

**파일 설명:** 회의실 예약 화면에서 사용자가 날짜를 선택했을 때, 이미 예약된 시간대를 실시간(AJAX)으로 조회하여 반환하는 기능을 담당합니다. 조회된 시간 목록을 JSON 형식의 문자열로 직접 조립하여 응답합니다.

# EqDeleteController.java

**파일 설명:** 관리자 페이지에서 비품 폐기(삭제) 요청을 처리합니다. 데이터베이스 제약 조건에 의해 대여 기록이 남아 있는 비품은 삭제되지 않도록 예외 처리가 되어 있어 시스템 안정성을 높였습니다.

# EqInsertController.java

**파일 설명:** 관리자가 새로운 비품 정보를 입력했을 때 이를 데이터베이스에 등록하는 역할을 합니다. 입력받은 비품명과 수량을 DTO에 담아 DAO로 전달하며, 등록 후 다시 관리자 목록 페이지로 안내합니다.

# EquipmentListController.java

**파일 설명:** 일반 사용자가 조회할 수 있는 전체 비품 목록을 불러오는 컨트롤러입니다. 데이터베이스에서 현재 등록된 모든 비품 정보를 가져와 화면에 전달하며, 사용자는 이를 통해 대여 가능한 비품 현황을 파악할 수 있습니다.

# EqUpdateController.java

**파일 설명:** 기존에 등록된 비품의 정보를 수정하는 기능을 담당합니다. 비품의 이름, 전체 수량, 현재 잔여 수량을 입력받아 데이터베이스를 업데이트하며, 관리자가 비품의 재고 상태를 실시간으로 조정할 수 있게 합니다.

# JoinController.java

**파일 설명:** 새로운 사원 계정을 생성하는 회원가입 로직을 처리합니다. 사번, 아이디, 비밀번호, 이름, 권한 레벨을 입력받아 저장하며, 사번 중복이나 데이터 형식 오류(숫자 미입력 등)에 대한 예외 처리가 포함되어 있습니다.

# LoginController.java

**파일 설명:** 사용자가 입력한 아이디(사번)와 비밀번호를 바탕으로 본인 인증을 수행합니다. 인증에 성공하면 사원 정보(DTO)를 세션에 저장하여 시스템 이용 권한을 부여하고 메인 화면으로 안내합니다.

# LogoutController.java

**파일 설명:** 현재 사용자의 세션 정보를 무효화하여 로그아웃 처리를 수행합니다. 세션을 완전히 삭제함으로써 보안을 유지하고, 다시 초기 로그인 화면으로 사용자를 되돌려보냅니다.

# ManagerApprovalController.java

**파일 설명:** 관리자 전용 결재함 목록을 구성합니다. 현재 로그인한 관리자의 권한 등급을 확인하여, 해당 등급에서 승인해야 할 '승인대기' 상태의 기안 문서들만 필터링하여 화면에 보여줍니다.

# MyRentalListController.java

**파일 설명:** 로그인한 사원 본인의 비품 대여 내역을 조회하는 기능을 담당합니다. 세션에서 사원 번호를 추출하여 해당 사원의 대여 기록만 데이터베이스에서 필터링해 가져오며, 비품 이름이 포함된 상세 리스트를 화면에 전달합니다.

# MyReserveController.java

**파일 설명:** 로그인한 사원이 예약한 회의실 목록을 보여주는 컨트롤러입니다. 대여 내역과 마찬가지로 세션 기반의 본인 확인을 거친 뒤, 특정 사원의 예약 데이터만 추출하여 시간순으로 정렬된 리스트를 제공합니다.

# RentFormController.java

**파일 설명:** 사용자가 비품 대여를 위해 신청 버튼을 눌렀을 때, 신청서 양식 페이지를 띄워주는 역할을 합니다. 선택한 비품의 상세 정보를 DB에서 실시간으로 가져와 화면에 미리 세팅함으로써 사용자가 정확한 비품을 선택했는지 확인할 수 있게 돕습니다.

# RentProcessController.java

**파일 설명:** 비품 대여 신청서에서 작성된 데이터를 받아 최종적으로 결재를 상신하는 역할을 합니다. 특히 우리 시스템의 특징인 '5단계 결재'의 시작점으로, 기안자가 서명함과 동시에 다음 결재자(2단계)에게 기안을 넘기는 로직이 포함되어 있습니다.

# ReserveController.java

**파일 설명:** 메인 화면의 층별 안내도에서 사용자가 특정 회의실을 클릭했을 때 호출됩니다. 클릭한 방의 번호를 전달받아 해당 회의실의 상세 스펙(수용 인원, 빔프로젝터 여부 등)을 DB에서 조회한 후 예약 신청 페이지로 연결해주는 가교 역할을 합니다.

# ReserveProcessController.java

**파일 설명:** 회의실 예약 신청 정보를 최종적으로 DB에 등록하는 컨트롤러입니다. 단순히 저장만 하는 것이 아니라, 같은 시간대에 이미 예약된 정보가 있는지 '중복 검사'를 수행하여 예약 겹침 현상을 원천 차단하는 로직이 핵심입니다.

# ReturnProcessController.java

**파일 설명:** 사용자가 대여 중인 비품을 반납할 때 실행되는 컨트롤러입니다. 특정 대여 번호를 받아 데이터베이스에서 해당 기록의 상태를 '반납완료'로 변경하고 처리 결과를 알림창으로 안내합니다.

**파일 설명:** 비품 대여 및 결재 시스템의 핵심 데이터 처리를 담당합니다. 신규 대여 신청(상신), 관리자 등급별 결재 대기 목록 조회, 5단계 순차 결재 처리, 그리고 사용자별 대여 이력 조회 기능을 포함하고 있습니다.

# ReservationDAO.java

**파일 설명:** 회의실 예약 정보를 관리합니다. 중복 예약 방지(checkDuplicate), 예약 시간 목록 조회(AJAX 연동용), 예약 취소 기능을 포함하며, 예약 등록 시에는 동시성 문제를 방지하기 위해 `synchronized` 키워드를 적용했습니다.

# RoomDAO.java

**파일 설명:** 회의실 마스터 정보(ROOM 테이블) 조회를 담당합니다. 사용자가 메인 지도의 특정 방을 클릭했을 때 해당 회의실의 상세 스펙(이름, 수용인원, 장비 유무 등)을 가져와 예약 화면에 세팅하는 기능을 제공합니다.

# #DTO

# EmployeeDTO.java

**파일 설명:** 시스템을 이용하는 사원의 정보를 담는 객체입니다. 로그인 인증 정보뿐만 아니라, 우리 프로젝트의 핵심인 '5단계 결재 시스템'의 권한을 결정하는 `empLevel` 필드를 포함하고 있습니다.

# EquipmentDTO.java

**파일 설명:** 사내에서 관리하는 비품(노트북, 마우스 등)의 마스터 데이터를 담고 있습니다. 전체 보유 수량과 현재 대여 가능한 잔여 수량을 관리하여 실시간 재고 파악의 기준이 됩니다.

# RentalHistoryDTO.java

**파일 설명:** 비품 대여 신청부터 반납까지의 모든 이력을 기록합니다. 특히 결재 단계(`approvalStep`)와 5명의 결재자 서명란(`sign1`~`sign5`) 필드가 있어, 기안이 현재 어느 단계에 머물러 있는지 상세히 추적할 수 있습니다.

# ReservationDTO.java

**파일 설명:** 회의실 예약 정보를 담는 객체입니다. 예약한 방, 날짜, 시간대(시작/종료)와 사용 목적을 관리하며, 중복 예약 방지 로직의 핵심 데이터로 사용됩니다.

# RoomDTO.java

**파일 설명:** 회의실 자체의 물리적인 스펙을 저장합니다. 수용 인원과 빔 프로젝터 구비 여부(`hasBeam`) 등의 정보를 담고 있어, 사용자가 본인의 용도에 맞는 방을 선택할 수 있도록 돕습니다.

# #View

# adminEqList.jsp

**파일 설명:** 관리자 전용 비품 재고 관리 대시보드 화면입니다. 현재 보유 중인 모든 비품의 리스트를 보여주며, 신규 비품 등록, 기존 정보 수정(모달창 이용), 그리고 비품 폐기(삭제) 기능을 한 화면에서 모두 처리할 수 있는 관리 도구입니다.

# equipmentList.jsp

**파일 설명:** 일반 사용자가 사내 비품을 대여하기 위해 목록을 확인하는 페이지입니다. 각 비품을 카드(Card) 형태로 보여주며, 실시간으로 재고(잔여 수량)를 확인하여 수량이 남아있는 경우에만 '대여 신청하기' 버튼이 활성화되도록 구현되어 있습니다.

# index.jsp

**파일 설명:** 시스템의 첫 화면인 로그인 페이지입니다. 사용자가 사번과 비밀번호를 입력하여 접속할 수 있는 폼을 제공하며, 계정이 없는 사용자를 위해 회원가입 페이지로 이동하는 버튼이 포함되어 있습니다.

# join.jsp

**파일 설명:** 새로운 사원이 계정을 생성하는 회원가입 페이지입니다. 사번(숫자), 아이디, 비밀번호, 이름, 그리고 결재 시스템에서 핵심이 되는 '직급(권한 레벨)'을 선택하여 가입을 요청할 수 있습니다.

# main.jsp

**파일 설명:** 시스템의 메인 대시보드 페이지입니다. 세션 정보를 확인하여 로그인하지 않은 사용자를 차단하며, 사용자의 권한 레벨(`empLevel`)에 따라 관리자 메뉴(결재함, 재고 관리)를 동적으로 보여줍니다. 층별 안내도 이미지를 배경으로 하여 클릭 가능한 회의실 예약 버튼들을 배치했습니다.

# managerApproval.jsp

**파일 설명:** 관리자가 자신에게 온 비품 대여 기안을 확인하고 승인 또는 반려를 결정하는 화면입니다. 특히 우리 시스템의 특징인 **5단계 결재 현황**을 시각적으로 보여주며, 각 단계별로 서명 여부에 따라 'O' 또는 'X' 표시와 녹색 배경(sign-done)이 적용됩니다.

# myRentalList.jsp

**파일 설명:** 로그인한 사용자가 자신의 비품 대여 내역과 결재 상태(승인대기, 대여중, 반려 등)를 한눈에 확인할 수 있는 페이지입니다. 현재 '대여중'인 비품에 대해서는 즉시 반납 절차를 진행할 수 있는 버튼이 활성화됩니다.

# myReserveList.jsp

**파일 설명:** 사용자가 신청한 회의실 예약 내역을 조회하는 화면입니다. 예약한 방, 날짜, 시간, 목적 등을 확인할 수 있으며, 아직 사용 시간이 되지 않은 **'예약완료'** 상태인 건에 대해서는 즉시 취소할 수 있는 기능을 제공합니다.

# rentForm.jsp

**파일 설명:** 사용자가 선택한 특정 비품을 대여하기 위해 대여 기간(시작일, 반납일)을 입력하고 결재를 요청하는 신청서 화면입니다. 비품 정보와 신청자 정보가 자동으로 표시되며, 자바스크립트를 통해 대여일과 반납일의 논리적 선후 관계를 검증하여 잘못된 데이터 입력을 방지합니다.

# reserve.jsp

**파일 설명:** 특정 회의실의 상세 정보(수용 인원, 기자재 유무)를 확인하고 예약을 신청하는 화면입니다. 그리드 형태의 시간 버튼 시스템을 통해 이용 시간을 직관적으로 선택할 수 있으며, **AJAX(Fetch API)** 기술을 사용하여 선택한 날짜에 이미 예약된 시간대를 실시간으로 조회해 중복된 슬롯을 비활성화하는 지능형 기능을 포함합니다.
